# 🚀 [추억여행 스케줄러] 8주 집중 개발 로드맵 (상세판)

## 🛠️ Phase 1: 기반 인프라 및 아키텍처 설계 (Week 1 - 2)
안정적인 서비스 운영을 위해 초반 데이터베이스 스키마와 API 통신망을 탄탄하게 구축하는 단계입니다.

* **Week 1: DB 및 백엔드 환경 세팅**
    * **DB 스키마 설계 (PostgreSQL):** 유저 정보, 방문 기록(방명록), 관광지 캐싱 테이블을 체계적으로 설계합니다.
    * **PostGIS 도입:** 공간 인덱스(Spatial Index)를 활용한 빠르고 효율적인 반경 검색을 위해 PostGIS 확장 모듈을 활용합니다.
    * **FastAPI 초기화:** 여러 외부 API를 효율적으로 병렬 호출할 수 있도록 비동기(`asyncio`) 환경으로 프로젝트를 구성합니다.
    * **API Key 발급 및 검증:** 카카오, 한국관광공사, 문화재청, 국가기록원 API 키를 발급받고 정상 호출 여부를 미리 꼼꼼하게 테스트합니다.

* **Week 2: 프론트엔드 기틀 및 연동 테스트**
    * **Flutter 프로젝트 세팅:** 상태 관리 패키지(Riverpod 등)를 도입하여 안정적인 디렉토리 구조와 아키텍처를 설정합니다.
    * **지도 렌더링:** 카카오맵 API를 연동하여 사용자 위치 기반으로 지도를 띄우고 커스텀 마커의 정상 작동 여부를 확인합니다.
    * **인증 로직:** 카카오 로그인 API를 통해 사용자 인증 처리를 진행하고, 유저 정보를 데이터베이스에 안전하게 저장하는 과정을 구현합니다.

---

## 📡 Phase 2: 코어 파이프라인 (데이터 수집 및 정제) (Week 3 - 4)
서비스의 핵심 가치인 맞춤형 관광 데이터를 수집하고 사용자가 보기 좋게 정제하는 단계입니다.

* **Week 3: 좌표 변환 및 반경 탐색 모듈**
    * **추억 좌표 변환 로직:** 사용자가 입력한 흐릿하거나 다양한 형태의 장소명을 카카오 로컬 및 검색 API를 활용해 정확한 위경도 좌표로 변환합니다.
    * **주변 인프라 수집:** 변환된 좌표를 중심으로 한국관광공사 API(`locationBasedList`)를 호출하여 반경 3~5km 내의 관광 데이터를 일괄 수집합니다.
    * **카테고리 분류:** 수집된 데이터를 기준(`categoryCode`, `contentTypeId`)에 따라 관광지, 음식점, 카페, 숙박 등으로 명확하게 분류합니다.

* **Week 4: 데이터 검증 및 보강 모듈**
    * **휴무일/영업시간 필터링:** 사용자에게 정확하고 실행 가능한 일정을 제공하기 위해, 휴무일과 영업시간을 확인하여 실제 방문 가능한 장소만 선별합니다.
    * **카드 UI용 데이터 보강:** 화면에 예쁘게 보여질 이름, 주소, 개요, 대표 이미지 등의 세부 정보를 추가로 수집하여 일정표의 완성도를 높입니다.

---

## 🧠 Phase 3: AI 두뇌 이식 및 스케줄링 (Week 5 - 6)
수집된 데이터를 바탕으로 AI 모델이 최적의 여행 일정을 지능적으로 구성하는 단계입니다.

* **Week 5: 동선 최적화 및 AI 프롬프트 엔지니어링**
    * **이동 시간 매트릭스:** 카카오 모빌리티 API를 활용해 선별된 장소들 간의 실제 차량/도보 이동 시간과 거리를 정확히 계산합니다.
    * **Gemma4-31b-it 연동:** 구글 스튜디오를 통해 고성능 언어 모델을 서비스에 연결합니다.
    * **프롬프트 템플릿 제작:** 장소 리스트, 이동 시간, 영업시간 데이터를 AI에 전달하여 당일치기 또는 1박 2일 일정을 구성하도록 시스템 프롬프트를 세밀하게 조정합니다. (결과물은 백엔드에서 처리하기 쉽도록 JSON 포맷으로 엄격하게 제한합니다.)

* **Week 6: 스케줄러 UI/UX 개발 (Flutter)**
    * **타임라인 뷰:** 백엔드에서 전달받은 JSON 스케줄 데이터를 사용자가 직관적으로 보기 편한 타임라인 형태의 UI로 세련되게 구현합니다.
    * **동선 렌더링:** 생성된 일정을 카카오맵 위에 선(Polyline)으로 연결하여 효율적인 이동 경로를 시각적으로 제공합니다.

---

## 🎨 Phase 4: 차별화 엣지 및 서비스 마감 (Week 7 - 8)
서비스의 감성적인 요소를 더하고 전반적인 품질과 성능을 끌어올리는 마무리 단계입니다.

* **Week 7: 과거 아카이브 및 소셜 기능 (방명록)**
    * **과거-현재 대조 기능:** 문화재청 및 국가기록원 API를 연동하여 해당 장소의 과거 사진을 불러오고, 현재 모습과 나란히 비교할 수 있는 감성적인 뷰를 개발합니다.
    * **디지털 방명록 개발:** 사용자가 특정 좌표에 마커를 남겨 추억을 기록하고, 이를 다른 사용자와 따뜻하게 공유할 수 있는 커뮤니티 기능을 구현합니다.

* **Week 8: 성능 최적화 및 QA (개선 작업)**
    * **API 캐싱 전략 적용:** 외부 공공 API의 응답 지연을 방지하기 위해, 한 번 검색된 좌표 주변의 데이터는 데이터베이스나 Redis에 캐싱하여 두 번째 요청부터는 쾌적하게 제공할 수 있도록 개선합니다.
    * **메모리 최적화:** Flutter 지도 위에서 다수의 마커와 동선을 처리할 때 발생할 수 있는 메모리 누수 문제를 철저하게 테스트하고 최적화하여 앱의 안정성을 확보합니다.
